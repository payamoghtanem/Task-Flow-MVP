# Procedures Directory

## Role
Standard Operating Procedures (SOPs) for recurring team processes. Every procedure is a step-by-step executable runbook with clear inputs, outputs, agents, and decision points.

## Scope
- In: Code review SOP, release process SOP, incident response SOP
- Out: Agent definitions, standards content, backlog management

## Write Authority
Orchestrator (01) writes. Scrum Master (03) approves. Lesson Learner (21) proposes updates.

## Procedures Index
- `code-review-process/CLAUDE.md` — Code review end-to-end
- `release-process/CLAUDE.md` — Release pipeline and sign-off
- `incident-process/CLAUDE.md` — Incident detection, triage, resolution

## Constraints
- Procedures are followed exactly — no improvisation
- Deviations require Orchestrator (01) + Scrum Master (03) approval and must be documented
- All procedures cite the standards versions they enforce
